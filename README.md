# 辽宁道路事故处理导航

一个面向普通事故当事人的移动端流程导航原型。它不要求用户先理解法律术语，而是通过已经发生的事实判断当前阶段，只展示此刻需要办理的事情、官方正在处理的事情、最近期限和材料缺口。

在线访问：<https://boboloveseggs.github.io/liaoning-accident-process-guide/>

## 核心能力

- 先选择“谁撞谁”和“我是哪一方”，再用事实问答自动定位当前阶段。
- 同时显示一个主阶段和一个并行事项，允许责任认定、治疗、取证同步进行。
- 页面主体固定为“我现在应该干什么”和“别人／官方正在干什么”两块。
- 责任认定、保险核定和复核期限先判断起算条件，再显示已过时间、最晚日期和当前状态。
- 材料按现场、治疗、赔偿阶段切换，并记录“已准备、已提交、机构已确认收到”三层状态。
- 首页集中显示今日必须做、下一步、最近期限、缺失材料和风险提醒。
- 提供异常情形提醒、官方沟通话术、规则来源、适用条件和核验日期。
- 所有个人进度只保存在当前设备的浏览器 `localStorage` 中，不上传服务器。

## 使用流程

1. 选择事故组合。
2. 选择自己的身份。
3. 回答最多四个事实问题，自动定位阶段。
4. 先看首页摘要和两块主体建议。
5. 按需展开“还要等多久”“具体怎么做”“我的材料”“为什么这样”“联系官方机构”。

## 当前阶段

1. 事故刚发生
2. 交警到场
3. 已离开现场
4. 等待责任认定
5. 治疗与赔偿
6. 争议或结案

## 本地运行

这是一个无构建步骤的单文件网页：

```powershell
cd legal-process-map-liaoning-v0.7.2
python -m http.server 9100
```

然后打开：

```text
http://127.0.0.1:9100/prototype/perspective-flow-v0.7.2.html
```
